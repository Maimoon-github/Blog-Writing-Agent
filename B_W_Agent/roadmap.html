config.py

Single source of truth for all tunables. Import constants here — never hardcode values in agents.

  • Ollama base URL, model name (mistral:7b)
  • SearxNG host (http://localhost:8080)
  • SD model ID, GPU flag, step/resolution defaults
  • Output directories, cache TTL (86400s)
schemas.py

All Pydantic models used as inter-agent contracts. Changing a schema here propagates everywhere.

  • Section, BlogPlan — planner output
  • ResearchResult — researcher output
  • SectionDraft — writer/editor output
  • ImageResult — image agent output
state.py

Defines CrewState TypedDict. This is the sole object that flows through the entire pipeline — every agent reads from and writes to it.

  • Input: topic: str
  • Router: research_required: bool
  • Workers: research_results, completed_sections, generated_images
  • Reducer: final_markdown, final_html, output_path
graph.py

Orchestration entry point. Chains the supervisor layer sequentially, then uses asyncio.gather to fire all worker crews in parallel, then runs Citation Manager → Reducer.

  • Imports and calls all 8 agent functions
  • Manages CrewState lifecycle
  • Wraps each worker in try/except with 3-tier fallback
  • SQLite checkpoint after each tier
streamlit_app.py

UI layer. Calls graph.py with the entered topic, streams progress, and renders the final blog preview with download links.

  • Topic text input + submit button
  • Per-agent progress status display
  • Rendered Markdown blog preview
  • Download buttons for .md and .html
tools/search.py

Wraps SearxNG + DuckDuckGo with a 24-hour disk-based JSON cache keyed by query hash. If cached, bypasses the network entirely.

  • search(query) → list of result dicts
  • Auto-fallback to DuckDuckGo on SearxNG failure
  • Cache stored in .cache/ per query hash
tools/image_gen.py

Stable Diffusion v1.4 pipeline wrapper. Detects CUDA availability at import time and sets appropriate resolution and step count.

  • GPU: 512×512, 50 steps, torch.float16
  • CPU: 384×384, 15 steps, torch.float32
  • Runs in ThreadPoolExecutor to stay off main event loop

Scaffold the directory structure
Create every folder and every empty file first. This lets you wire imports and test the module graph before any logic exists.

Write config.py first
Every other file imports from here. Centralise all tunables so you never hardcode a value in an agent file.
# config.py — key constants OLLAMA_BASE_URL = "http://localhost:11434" OLLAMA_MODEL = "mistral:7b" SEARXNG_HOST = "http://localhost:8080" SD_MODEL_ID = "CompVis/stable-diffusion-v1-4" USE_GPU = True # set False for CPU-only IMG_SIZE = (512, 512) # GPU; (384,384) on CPU IMG_STEPS = 50 # GPU; 15 on CPU CACHE_TTL_SEC = 86400 OUTPUT_BLOGS = "outputs/blogs" OUTPUT_IMAGES = "outputs/images"
3
Define all Pydantic schemas in schemas.py
These are the inter-agent contracts. Write them all before any agent logic — they enforce the data shapes that every agent must produce and consume.
from pydantic import BaseModel from typing import List, Dict, Optional, Tuple from datetime import datetime class Section(BaseModel): id: str title: str description: str word_count: int search_query: Optional[str] image_prompt: str class BlogPlan(BaseModel): blog_title: str feature_image_prompt: str sections: List[Section] research_required: bool class ResearchResult(BaseModel): section_id: str summary: str sources: List[Dict[str, str]] timestamp: datetime class SectionDraft(BaseModel): section_id: str title: str content: str # Markdown with [citation] and [IMAGE_PLACEHOLDER_{id}] word_count: int citations: List[str] class ImageResult(BaseModel): section_id: str prompt: str file_path: str alt_text: str size: Tuple[int, int]
4
Define CrewState in state.py
The single shared TypedDict that flows through the entire pipeline. Every agent receives this as input and returns a partial update dict.
from typing import TypedDict, List, Optional from schemas import BlogPlan, ResearchResult, SectionDraft, ImageResult class CrewState(TypedDict): topic: str research_required: bool plan: Optional[BlogPlan] research_results: List[ResearchResult] completed_sections: List[SectionDraft] generated_images: List[ImageResult] citation_registry: dict final_markdown: str final_html: str output_path: str

Write the prompt .txt files
Populate each prompt template with detailed instructions. Storing them as text files means you can tune prompts without touching Python code. Keep prompts in second person ("You are a...") and include explicit output format instructions. The planner prompt is the most critical — write it last after you've tested the other agents.

Phase 3 — Agent Logic Implementation
Implement files in dependency order — tools first, then agents, then orchestrator
Implementation order matters. Build in this sequence: tools/ → agents/citation_manager.py → agents/reducer.py → agents/router.py → agents/planner.py → agents/researcher.py → agents/writer.py → agents/editor.py → agents/image_agent.py → graph.py → streamlit_app.py. Start with the nodes that have no dependencies and build upward.
Step 1 tools/search.py — Search wrapper with caching
›
Cache layer first
Implement a cache_get(query) / cache_set(query, results) pair that stores JSON in .cache/{hash}.json with a TTL check. This prevents redundant network calls during development.
›
SearxNG integration
Use langchain_community.utilities.SearxSearchWrapper(searx_host=SEARXNG_HOST). Wrap in try/except — on any failure, fall through to the DuckDuckGo fallback. Return a list of {"url": str, "title": str, "snippet": str} dicts.
›
DuckDuckGo fallback
Import DuckDuckGoSearchRun from langchain_community.tools. If SearxNG raises, call DuckDuckGo with the same query. Normalise the output to the same dict shape.
zero dependenciesimplement + test first
Step 2 tools/image_gen.py — Stable Diffusion pipeline
›
GPU/CPU detection at module load time
Check torch.cuda.is_available() once at import. Set dtype, device, image size, and step count accordingly. Load the pipeline onto the device immediately — don't defer to the first call.
›
generate_image(prompt, section_id) → str
Run the SD pipeline, save the image to outputs/images/{section_id}.png using a deterministic filename, and return the file path. This function will be called inside a ThreadPoolExecutor from the orchestrator.
ThreadPoolExecutordeterministic filenames
Step 3 agents/citation_manager.py — Pure Python, no LLM
›
Build the citation registry
Aggregate all ResearchResult.sources from CrewState.research_results. Deduplicate by URL. Assign sequential numbers: {1: {url, title, snippet}, 2: ...}.
›
Resolve inline markers
Use regex to find [citation] markers in each section draft's content. Replace with [N] where N maps to the registry. Also append a formatted "References" section at the end. Return updated completed_sections and the citation_registry.
Step 4 agents/reducer.py — Final assembler
›
Order sections by plan index
Sort completed_sections by matching section_id to the order in plan.sections. Never assume they arrive in order from parallel workers.
›
Substitute image placeholder tokens
For each [IMAGE_PLACEHOLDER_{id}] token in a section's content, look up the matching ImageResult and replace with ![alt_text](file_path). If no image exists for that ID, replace with an empty paragraph.
›
Write .md and .html output files
Assemble the full Markdown string: metadata block → feature image → sections → references. Write to outputs/blogs/{slug}.md. Convert to HTML using the markdown library and write .html. Store both paths in CrewState.
implement before routertestable independently
Step 5 agents/router.py — Intent classifier
›
Load prompt and build LLM chain
Read prompts/router_prompt.txt. Create a ChatOllama instance with temperature=0 (deterministic for classification). Format as a ChatPromptTemplate with {topic} as the only variable.
›
router_node(state) → dict
Invoke the chain. Parse the response for a true/false classification. Return {"research_required": bool}. Wrap in try/except — on failure, default to True (safe fallback: always research).
Step 6 agents/planner.py — BlogPlan generator
›
Structured output via Pydantic
Use llm.with_structured_output(BlogPlan) from LangChain to enforce the exact BlogPlan schema. This forces Mistral to output valid JSON that maps to the model. Retry up to 3 times if parsing fails.
›
dispatch_crews() factory
Implement the crew factory function that, given a BlogPlan, creates one CrewAI Crew per section for each worker type (Researcher, Writer, Editor, Image). Returns a flat list of crews to be gathered by the orchestrator. This is the key parallelism function.
›
ChromaDB RAG query (optional Phase 2 feature)
Before generating a new plan, query the ChromaDB "blog_research" collection with similarity_search(topic, k=3). If relevant past research exists, inject it into the planner prompt as additional context. This is an enhancement — wire up core logic first.
most complex agentimplement last in this group
Step 7 agents/researcher.py — Web research worker
›
researcher_node(section) → ResearchResult
Takes a single Section object. Call tools.search.search(section.search_query) 2–3 times with slightly varied queries. Pass top results + snippets to Mistral for summarization into 300–500 words. Return a ResearchResult.
›
3-tier fallback
Tier 1: retry 3× with exponential backoff (1s, 2s, 4s). Tier 2: on total failure, return an empty ResearchResult with summary="[RESEARCH_FAILED]" and empty sources. The Writer will handle this gracefully.
Step 8 agents/writer.py + agents/editor.py
›
writer_node(section, research) → SectionDraft
Load writer_prompt.txt. Inject: section description, target word count, research summary, source URLs. Invoke Mistral. Parse the Markdown response. Count words. Insert [IMAGE_PLACEHOLDER_{section.id}] after the first paragraph. Return a SectionDraft.
›
editor_node(draft) → SectionDraft
Load editor_prompt.txt. Pass the full draft content. Ask Mistral to improve clarity, SEO density, flow, and enforce word count. Return an updated SectionDraft with the same section_id. The editor must preserve all [IMAGE_PLACEHOLDER_...] and [citation] tokens.
both are parallel per sectioneditor waits on writer output
Step 9 agents/image_agent.py — Visual content creator
›
image_agent_node(section) → ImageResult
Call tools.image_gen.generate_image(section.image_prompt, section.id) in a ThreadPoolExecutor (keeps image generation off the async event loop). On failure, return an ImageResult with file_path="PLACEHOLDER" — the Reducer handles this gracefully by emitting plain text instead of an image tag.
Tier 3 skip-and-continue fallbackmost latency-sensitive
Step 10 graph.py — Orchestration pipeline
›
Sequential supervisor layer
Chain router_node → planner_node sequentially. After each, update the CrewState dict with the returned fields.
›
Parallel worker dispatch via asyncio.gather
Build coroutines for every Researcher, Writer, Editor, and Image crew. Call await asyncio.gather(*coroutines). Merge the results back into CrewState.research_results, .completed_sections, and .generated_images lists.
crews = dispatch_crews(state) results = await asyncio.gather(*[c.kickoff_async() for c in crews]) # merge results into state
›
Sequential termination layer
After gather completes, run citation_manager_node → reducer_node sequentially. Save SQLite checkpoint after each stage. Return final CrewState.
Step 11 streamlit_app.py — UI layer
›
Topic input form
Use st.text_input for the topic. A "Generate" button triggers the pipeline. Wrap the graph.py call in asyncio.run() since Streamlit is synchronous.
›
Progress display
Use st.status containers to show each agent phase: Routing → Planning → Researching (N sections) → Writing → Editing → Generating images → Assembling. Update these from a progress callback passed into graph.py.
›
Output preview + download
Use st.markdown to render the final blog inline. Provide st.download_button for both .md and .html outputs. Display generated images using st.image with the paths from CrewState.generated_images.

Phase 4 — Testing & Deployment
Validate each layer in isolation before integration testing the full pipeline
Test strategy Unit → Integration → End-to-End ~1 week
T1
Unit test each agent in isolation
For each agent, construct a minimal CrewState with just the fields that agent reads, call the node function, and assert the returned dict has the expected schema. Mock Ollama and SearxNG for speed.
from agents.router import router_node state = {"topic": "latest AI research 2025", "research_required": False} result = router_node(state) assert "research_required" in result assert isinstance(result["research_required"], bool)
T2
Test tools independently
Run tools/search.py with a real query against your running SearxNG Docker container. Verify the JSON cache file is written. Run again — verify it's served from cache (no network call). Test the DuckDuckGo fallback by stopping the SearxNG container.
requires running SearxNGrequires running Ollama
T3
Integration test: Researcher → Writer → Editor chain
Create one Section object, pass it through Researcher, then Writer, then Editor in sequence. Assert that the final SectionDraft contains an [IMAGE_PLACEHOLDER_...] token, has a non-zero word count, and citations list is populated.
T4
Integration test: Citation Manager → Reducer
Construct a mock CrewState with 2–3 fake SectionDraft objects and ResearchResult objects. Run Citation Manager and assert the registry is built. Pass to Reducer and assert the output .md file is written to disk with correct image and citation syntax.
T5
Full end-to-end pipeline test (no images)
Call graph.py directly with a simple, well-known topic (e.g., "What is machine learning"). Disable image generation (SKIP_IMAGES=True flag in config). Measure total wall time. Target: under 3 minutes for 4 sections. Check the output .md file for: all sections present, no orphaned [citation] markers, references section appended.
T6
Full end-to-end test with images
Re-enable image generation. Run the pipeline on a time-sensitive topic (e.g., "AI trends 2025") to exercise the research_required=True path. Verify: feature image + 4 section images generated, all placeholder tokens resolved in the HTML output, all citation URLs are real and accessible.
target: under 6 min on RTX 3060success metric from spec
T7
Test 3-tier fallback behaviour
Stop SearxNG. Run the pipeline — verify DuckDuckGo fallback fires. Stop Ollama mid-run — verify retry logic triggers. Use a malformed topic string — verify Router handles it gracefully without crashing the pipeline.
T8
Streamlit UI acceptance test
Run streamlit run streamlit_app.py. Enter a topic, click Generate, and verify: progress indicators update per phase, blog renders correctly in the preview pane, both .md and .html download buttons work, images display inline.
streamlit run streamlit_app.py --server.port 8501

D4
Performance profiling + optimization
Profile the pipeline with cProfile or LangSmith traces. Common bottlenecks: image generation (move to background thread), sequential SearxNG page fetches (batch with asyncio.gather), Mistral cold-start (keep Ollama warm between runs with a keep-alive ping).

✓ Definition of Done — v1 Release Checklist
  • End-to-end run completes in under 6 minutes on RTX 3060 for a 4-section blog post.
  • At least 5 images generated per post — 1 feature image + 1 per section — and all placeholder tokens resolved in output.
  • Zero hallucinated citations — every reference URL is a real URL extracted from SearxNG/DuckDuckGo results.
  • 3-tier fallback verified — pipeline completes even when SearxNG is down and when image generation fails.
  • Streamlit UI tested end-to-end — progress updates, blog preview renders, both download buttons work.
  • SQLite checkpoint tested — kill the process mid-run, restart, verify it resumes from the last completed agent tier.
  • Research cache verified — run the same topic twice, confirm second run skips network calls (under 60s total).
  • All outputs are gitignored — outputs/, chroma_db/, .cache/, __pycache__/ absent from git status.
